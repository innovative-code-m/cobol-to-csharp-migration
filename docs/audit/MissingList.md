# Missing List: COBOL to C# Migration Audit (v1)

現行のルール集 v1 では定義が不足しており、変換精度向上や運用設計のために追加・詳細化が必要な要素の一覧です。

| 未定義要素 | 重要度 | 理由 | 提案カテゴリ |
|:---|:---|:---|:---|
| **編集用PICの書式化** | **High** | `Z,ZZZ,ZZ9.99` 等の出力形式は、C#の標準的な `ToString()` 指定だけでは、ゼロサプレスや符号位置のCOBOL完全互換を担保できないため。 | v1対応に追加 |
| **File Statusコード対応** | **Med** | `FS-ORD NOT = "00"` 等のステータス判定。C#の例外処理と、COBOLが期待するステータスコードの相互マッピング定義が必要。 | v1対応に追加 |
| **プログラム終了制御** | **Med** | `STOP RUN` やエラー出力時の終了コード管理、およびバッチ全体の共通例外ハンドリング方針の定義。 | v1非対応/TODO |
| **SIGN句/符号位置** | **Low** | `PIC S9` 等の符号が、外部ファイル上で「独立項目」か「埋め込み」かによるバイナリ変換への影響。 | v1非対応/TODO |

---
**Note:** 重要度Highの項目は、帳票出力や画面表示の「見た目」に直結するため、次期バージョンでの優先的なルール化を推奨します。
